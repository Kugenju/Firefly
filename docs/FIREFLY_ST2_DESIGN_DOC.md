## 二、核心机制

### 1. 灼热 (Scorch) - 修改后
**机制描述**: 当带有灼热的敌人进行攻击意图时，立即受到对应层数的伤害，然后灼热层数减少1（每次攻击意图触发一次）。

**规则详情**:
- 触发时机: 敌人显示攻击意图时，在其行动前触发
- 伤害计算: 每层灼热造成1点伤害
- 衰减机制: 每次触发后层数-1
- 上限: 无上限，可无限叠加
- 视觉表现: 敌人身上带有火焰特效

**与游戏内机制的对比**:
- 类似Thorns（荆棘），但触发时机不同（攻击意图vs受到伤害）
- 类似Poison（毒），但触发时机不同（行动前vs回合开始）

---

### 2. 完全燃烧 (Complete Combustion)
**机制描述**: 流萤的强化形态，进入后获得全面提升。

**效果详情**:
- 持续时间: 3回合
- 攻击力加成: +3（所有攻击伤害+3）
- 额外抽牌: 每回合多抽1张牌
- 特殊卡牌: 可使用专属强化卡牌
- 退出条件: 持续回合结束自动退出

**视觉表现**: 
- 火焰从红色变为绿色
- 装甲展开特效
- 双剑武器形态

---

### 3. 生命消耗机制
**机制描述**: 部分卡牌消耗生命值而非/额外于能量。

**规则详情**:
- 生命值不足时，仍可打出（生命值降至1）
- 生命消耗不计入受到伤害（不触发受伤相关效果）

## 六、关键词定义

### 灼热 (Scorch)
- 带有灼热的敌人在显示攻击意图时，立即受到等于灼热层数的伤害。
- 触发后灼热层数减少1。
- 属于状态效果，可被清除。

### 完全燃烧 (Complete Combustion)
- 持续3回合的强化状态。
- 攻击力+3，每回合多抽1张牌。
- 可使用专属强化卡牌。

### 萤火护盾 (Flicker Shield)
- 抵消下一次受到的全部伤害。
- 持续1回合，触发后消失。

## 七、战斗循环设计

### 理想循环
1. **启动阶段**: 使用充能卡积累能量，适当消耗生命值
2. **爆发阶段**: 打出「完全燃烧」进入强化状态
3. **输出阶段**: 使用强化卡牌（如死星过载）打出高额伤害
4. **收尾阶段**: 通过灼热层数在敌人行动时造成额外伤害

### 风险控制
- 生命消耗需要谨慎，避免过度自损
- 完全燃烧有持续时间，需要规划好输出窗口
- 灼热需要持续叠加才能发挥效果

## 九、代码实现要点

### 灼热机制
```csharp
// 伪代码
public class ScorchPower : PowerModel {
    // 当敌人显示攻击意图时触发
    public override void OnEnemyIntentShown(Enemy enemy) {
        enemy.TakeDamage(Amount);
        Amount--; // 触发后层数-1
    }
}
```

### 完全燃烧状态
```csharp
public class CompleteCombustionPower : PowerModel {
    public override void OnTurnStart() {
        // 攻击力+3
        Owner.AddTemporaryStrength(3);
        // 额外抽牌
        Owner.DrawCards(1);
    }
}
```
