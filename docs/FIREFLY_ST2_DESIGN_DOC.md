## 一、角色概述

### 背景故事
流萤是格拉默铁骑的幸存者，编号AR-26710。作为战胜虫群的基因改造战士而生，她驾驶着战术装甲「萨姆」战斗。为了寻找「生」的机会，她加入星核猎手，在宇宙中追寻违抗命运的方式。

> "火萤这种生物很神奇吧？它们或许会扑向火烛，或许会突然老化。但在那之前的每一个夜晚，它们的光芒会比星星更耀眼。"

## 二、核心机制

### 1. 灼热 (Scorch)
**机制描述**: 当带有灼热的敌人进行攻击意图时，立即受到对应层数的伤害，然后灼热层数减少1（每次攻击意图触发一次）。

**规则详情**:
- 触发时机: 敌人显示攻击意图时，在其行动前触发
- 伤害计算: 每层灼热造成1点伤害
- 衰减机制: 每次触发后层数-1
- 上限: 无上限，可无限叠加
- 视觉表现: 敌人身上带有火焰特效

**玩家效果**: 玩家打出攻击牌时，受到等于灼热层数的伤害，然后层数减少1。

**与游戏内机制的对比**:
- 类似Thorns（荆棘），但触发时机不同（攻击意图vs受到伤害）
- 类似Poison（毒），但触发时机不同（行动前vs回合开始）

---

### 2. 萤火 (Firefly) - 新增机制 ⭐
**机制描述**: 流萤卡组的核心卡牌属性机制，类似猎手的奇巧牌。

**萤火状态**:
- **未激发**: 普通状态，正常效果，正常耗能
- **激发**: 卡牌效果翻倍，耗能-1（至少1点）

**触发条件**:
- 打出「完全燃烧」会激发所有手牌中的萤火牌
- 部分卡牌可以直接激发萤火牌

**视觉表现**:
- 未激发: 卡牌边框呈淡蓝色微光
- 激发: 卡牌边框燃烧绿色火焰，并有萤火虫粒子特效

**初始卡组**: 打击和防御**不**附带萤火属性

---

### 3. 完全燃烧 (Complete Combustion) - 重做 ⭐
**机制描述**: 流萤的强化形态，改为以萤火机制为核心。

**效果详情**:
- 持续时间: 3回合
- **激发所有萤火牌**: 手牌中所有萤火牌进入激发状态（效果翻倍，耗能-1）
- 额外抽牌: 每回合多抽1张牌
- 伤害减免: 受到的所有伤害-5
- 退出条件: 持续回合结束自动退出

**与旧版对比**:
| 项目 | 旧版 | 新版 |
|------|------|------|
| 攻击力加成 | +3 | 移除 |
| 萤火效果 | 无 | 激发所有萤火牌 |
| 防御能力 | 无 | 受到伤害-5 |
| 抽牌 | 每回合+1 | 每回合+1（保留） |
| 形态转换 | 打击/防御变身 | 移除（改为独立卡牌） |

---

### 4. 裂解 (Dissolution) - 新增机制 ⭐
**机制描述**: 模拟流萤在《崩坏：星穹铁道》中的「弱点击破」能力，将敌人的格挡视为「韧性条」。当格挡被击破时，造成反馈伤害。

**规则详情**:
- **触发条件**: 敌人的格挡从有到无被击破时触发（从>0变为=0）
- **伤害计算**: 受到等同于**本回合该敌人获得的所有格挡总数**的伤害
- **视觉表现**: 格挡破碎时产生绿色火焰/粒子特效（呼应完全燃烧的火焰）
- **标签效果**: 触发裂解伤害的敌人会带有「裂解」标记

**设计来源**:
- 流萤的「失熵症」设定（物理结构解离）
- 格拉默铁骑的毁灭性打击风格
- 「弱点击破」→「格挡击破」的概念迁移

---

### 5. 生命消耗机制
**机制描述**: 部分卡牌消耗生命值而非/额外于能量。

**规则详情**:
- 生命值不足时，仍可打出（生命值降至1）
- 生命消耗不计入受到伤害（不触发受伤相关效果）

## 六、关键词定义

### 灼热 (Scorch)
- 带有灼热的敌人在显示攻击意图时，立即受到等于灼热层数的伤害，然后层数减少1。
- 玩家打出攻击牌时，受到等于灼热层数的伤害，然后层数减少1。
- 属于状态效果，可被清除。

### 萤火 (Firefly) ⭐新增
- 卡牌的固有属性，类似猎手的「奇巧」。
- **未激发**: 正常效果，正常耗能。
- **激发**: 效果翻倍，耗能-1（至少1点）。
- 可通过「完全燃烧」、卡牌效果等方式激发。

### 完全燃烧 (Complete Combustion) ⭐重做
- 持续3回合的强化状态。
- **激发所有萤火牌**: 手牌中所有萤火牌进入激发状态。
- 每回合多抽1张牌。
- 受到的所有伤害-5。

### 萤火护盾 (Flicker Shield)
- 抵消下一次受到的全部伤害。
- 持续1回合，触发后消失。

### 裂解 (Dissolution) ⭐新增
- 当敌人的格挡从有到无被击破时触发。
- 敌人受到等同于本回合其获得的所有格挡总数的伤害。
- 触发裂解的敌人会带有「裂解」标记（用于后续卡牌效果）。

## 七、战斗循环设计

### 理想循环 - 萤火路线
1. **积累阶段**: 通过「萤火燎原」给关键牌添加萤火属性，或抽取自带萤火的卡牌
2. **激发阶段**: 打出「点燃大海」或「完全燃烧」激发萤火牌
3. **爆发阶段**: 使用「海一直燃」打出所有激发状态的萤火牌
4. **收尾阶段**: 利用激发状态的高效果牌清理战场

### 理想循环 - 灼热路线（保留）
1. **叠加阶段**: 使用灼热卡牌叠加层数
2. **控制阶段**: 利用灼热伤害在敌人行动前削弱
3. **爆发阶段**: 「完全燃烧」提供生存能力，继续叠加
4. **收割阶段**: 敌人行动时触发大量灼热伤害

### 三路线融合
- **灼热路线**: 持续施加灼热，在敌人行动前造成伤害
- **萤火路线**: 积累萤火牌，通过激发打出爆发伤害
- **裂解路线**: 操控敌人格挡，击破时造成高额反馈伤害

三路线之间可相互联动：
- 萤火牌可以施加灼热（如闪燃打击）
- 裂解可以触发灼热叠加（母虫之怒）
- 「完全燃烧」同时服务于三条路线

### 萤火卡牌实现
```csharp
[Pool(typeof(FireflyCardPool))]
public class FlashBurnStrike : CardModel
{
    public FlashBurnStrike() : base(1, CardType.Attack, CardRarity.Uncommon, TargetType.AnyEnemy, false)
    {
        // 添加萤火关键词
        Keywords.Add(FireflyKeywords.Firefly);
    }
    
    protected override IEnumerable<DynamicVar> CanonicalVars => new[]
    {
        new DamageVar(8m, ValueProp.Move)
    };
    
    // 判断是否被激发
    private bool IsIgnited => this.GetModifier<FireflyIgniteModifier>()?.IsIgnited ?? false;
    
    protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
    {
        if (cardPlay.Target != null)
        {
            // 如果被激发，效果翻倍
            decimal damage = IsIgnited ? DynamicVars.Damage.BaseValue * 2 : DynamicVars.Damage.BaseValue;
            int scorch = IsIgnited ? 4 : 2;
            
            await PowerCmd.Apply<ScorchPower>(cardPlay.Target, scorch, Owner?.Creature, this);
            
            await DamageCmd.Attack(damage)
                .FromCard(this)
                .Targeting(cardPlay.Target)
                .Execute(choiceContext);
        }
    }
}
```

### 完全燃烧新效果
```csharp
public class CompleteCombustionPower : PowerModel
{
    public override void OnTurnStart()
    {
        // 激发所有手牌中的萤火牌
        foreach (var card in Owner.HandPile.Cards)
        {
            if (card.Keywords.Contains(FireflyKeywords.Firefly))
            {
                card.GetOrAddModifier<FireflyIgniteModifier>().IsIgnited = true;
            }
        }
        // 额外抽牌
        Owner.DrawCards(1);
    }
    
    public override int ModifyDamageReceived(int damage)
    {
        return damage - 5; // 伤害减免
    }
}
```
